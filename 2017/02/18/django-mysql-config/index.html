Django+MySQL 配置

发表于 2017-02-18 | 分类于 折腾 | | 阅读次数

在 Mac OS 下配置 Django 和 MySQL。

安装环境

操作系统：macOS Sierra [10.12]
开发环境：Python 2.7.13、MySQL 5.5
安装：
Django(1.10.5): pip install Django (查看版本号：python -m django --version)
MySQL-python(1.2.5):
brew install mysql-connector-c
pip install MySQL-python

文件说明
详细解释：

  • 外层的demo/目录与Django无关，只是你项目的容器，可以任意命名。
  • manage.py：一个命令行工具，用于与Django进行不同方式的交互脚本，非常重要！
  • 内层的demo/目录是真正的项目文件包裹目录，它的名字是你引用内部文件的包名，例如：demo.urls。
  • demo/__init__.py:一个定义包的空文件。
  • domo/settings.py:项目的主配置文件，非常重要！
  • demo/urls.py:路由文件，所有的任务都是从这里开始分配，相当于Django驱动站点的内容表格，非常重要！
  • demo/wsgi.py:一个基于WSGI的web服务器进入点，提供底层的网络通信功能，通常不用关心。

开发服务器

cd demo 进入 demo/ 目录，输入命令：
python manage.py runserver
将看到下面的提示：

Django提供了一个用于开发的web服务器，无需配置线上服务器，就能让站点运行起来。
这是一个简易的测试服务器。在浏览器访问 http://127.0.0.1:8000/ ，将看到Django的欢迎界面。

django开发服务器默认运行在内部的8000端口，如果希望指定端口，请在命令中显示给出，例如：
python manage.py runserver 8080
运行后Django将运行在8080端口。
如果想指定服务器的IP，将其与端口一并给出：
python manage.py runserver 0.0.0.0:8000
整个子网内都将可以访问，而不只本机。
注：Django的开发服务器具有自动重载功能，当代码有修改，每隔一段时间服务器将自动更新。但是，有一些例如增加文件的动作，不会触发服务器重载，这时就需要手动重启。

创建应用

app与project的区别：

  • 一个app实现某个功能，比如博客、公共档案数据库或者简单的投票系统；
  • 一个project是配置文件和多个app的集合，他们组合成整个站点；
  • 一个project可以包含多个app；
  • 一个app可以属于多个project.

app的存放位置可以是任何地点，但是通常我们将它们都放在与manage.py同级目录下，这样方便导入文件。

修改 settings.py

  1. 将 demoapp 加入到 INSTALLED_APPS

  2. 将默认的sqlite数据库改成mysql

  3. 建议将 TIME_ZONE 设置为所在的时区[‘Asia/Shanghai’(‘PRC’)]。参考时区列表。

测试连接

在 demo/ 目录下输入：
python manage.py dbshell

vi ~/.bash_profile

export DYDL_LIBRARY_PATH="$DYDL_LIBRARY_PATH:/usr/local/mysql/lib"
在命令行输入：
ln -s /usr/local/mysql/bin/* /usr/local/bin

同步数据库

python manage.py migrate
migrate命令将遍历INSTALLED_APPS设置中的所有项目，在数据库中创建对应的表，并打印出每一条动作信息。

可以在数据库命令行下输入：SHOW TABLES; (MySQL), 或 .schema (SQLite) 来列出 Django 所创建的表。

对于极简主义者，完全可以在INSTALLED_APPS内注释掉任何或者全部的Django提供的通用应用。这样，migrate也不会创建对应的数据表。
