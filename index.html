GMM的EM算法实现

发表于 2016-10-28 |

matlab下混合高斯模型的EM算法实现。

参数说明

N：样本数
M：Gauss数

高斯模型

模型表示

$$N(\mathbf{x;\mu,\Sigma})=\frac{1}{(2\pi)^{\frac{d}{2}}|\mathbf{\Sigma}|^{\frac{1}{2}}\exp{-\frac{1}{2}(\mathbf{x-\mu})^T\mathbf{\Sigma}^{-1}(\mathbf{x-\mu})}}$$
其中，d 为维数。

最大似然估计

$$L(\theta)=\sum_{n=1}^N\log N(\mathbf{x}_n;\mathbf{\mu,\Sigma})=-\frac{1}{2}\sum_{n=1}^N{d\log(2\pi)+\log|\mathbf{\Sigma}|+(\mathbf{x}_n-\mathbf{\mu})^T\mathbf{\Sigma}^{-1}(\mathbf{x}_n-\mathbf{\mu})}$$

得到的参数估计为：
$\mathbf{\hat\mu}=\frac{1}{N}\sum_{n=1}^N\mathbf{x}_n$

$\mathbf{\hat\Sigma}=\frac{1}{N}\sum_{n=1}^N(\mathbf{x}_n-\mathbf{\hat\mu})(\mathbf{x}_n-\mathbf{\hat\mu})^T$

混合高斯模型

模型表示

$p(\mathbf{x})=\sum_{m=1}^Mc_mN(\mathbf{x;\mu}_m,\mathbf{\Sigma}_m))$

$s.t. \sum_{m=1}^Mc_m=1$

最大似然估计

$L(\theta)=\sum_{n=1}^N\log\left(\sum_{m=1}^Mc_mN(\mathbf{x}_n;\mathbf{\mu}_m,\mathbf{\Sigma}_m)\right)$

$\gamma_m(n)=P(m;\mathbf{x_n},\hat\theta)=\frac{p(\mathbf{x}_n;m,\hat\theta)P(m;\hat\theta)}{\sum\limits_{k=1}^Mp(\mathbf{x}_n;k,\hat\theta)P(k;\hat\theta)}$

=K+\sum{n=1}^N\sum{m+1}^M\gamma_m(n)\log cm-\frac{1}{2}\sum{n=1}^N\sum_{m+1}^M\gamma_m(n)\left(\log|\mathbf{\Sigma}_m|+(\mathbf{x}_n-\mathbf{\mu}_m)^T\mathbf{\Sigma}_m^{-1}(\mathbf{x}_n-\mathbf{\mu}_m)\right))

EM算法步骤

  1. 初始化参数，设置迭代次数。假设只有单Gauss，估计其模型参数，。对于混合高斯模型中的M个Gauss，采用全局初始化参数，加一随机移动（使得M个Gauss不同，否则等同于单Gauss），作为它们的初始化参数。

  2. 迭代次数。对于每个样本,采用计算后验概率}(n)).

    }(n)=\frac{c_m^{(k-1)}N(\mathbf{x}_n;\mathbf{\mu}_m^{(k-1)},\mathbf{\Sigma}m^{(k-1)})}{\sum{j=1}^Mc_m^{(k-1)}N(\mathbf{x}_n;\mathbf{\mu}_j^{(k-1)},\mathbf{\Sigma}_j^{(k-1)})})

  3. 更新模型参数}={c_m^{(k)},\mathbf{\mu}_m^{(k)},\mathbf{\Sigma}_m^{(k)}})

LaTeX 把下标放在正下方

发表于 2016-10-24 |

在用$\LaTeX$输入数学公式时，有时需要将公式的下标放在正下方：

  1. 如果是数学符号，那么直接在_前加\limits命令放在正下方，如sum函数下面的取值范围，一般情况下是放在$\sum$的正下方。如果$\sum$在分母中，则默认会将取值范围置于右下角，如果需要放在正下方。可以如下实现：

    f(x,y)=\frac{mn}{\sum\limits_{(s,t)\in S_{xy}}{\frac{1}{g(x,y)}}}

    显示为 $$f(x,y)=\frac{mn}{\sum\limits_{(s,t)\in S_{xy}}{\frac{1}{g(x,y)}}}$$

  2. 若是普通符号，那么要用\mathop先转成数学符号再用\limits，如

    f(x,y)=\mathop{medium}\limits_{(s,t)\in S_{xy}}{g(x,y)})

    显示为 $$f(x,y)=\mathop{medium}\limits_{(s,t)\in S_{xy}}{g(x,y)}​$$

  3. 对于双美元符之间的公式，Latex默认下标是放在正下方。所以就不需要加\limits。

MarkDown 图片编辑

发表于 2016-09-26 |

插入图片

本地图片

MarkDown中插入本地图片的语法为：![](dir/imgname)

  1. 使用相对路径插入图片
    如，把名为1.png的图片和.md文件放在一起，则可以插入图片![](1.png)
    如果1.png图片放在与    .md同一目录的名为image的文件夹下，则可以插入图片![](image/1.png)

  2. 同样地，图片路径可采用绝对路径

利用GitHub

利用Github存储图片，在*.md中引用图片链接地址。采用如下步骤：

  1. 将所需的图片发布到GitHub上
  2. 访问GitHub上的图片 Image_Example
  3. 右键 Download，选择复制链接。
  4. 在*.md中引用图片![](img_url),其中img_url为所复制的链接。
